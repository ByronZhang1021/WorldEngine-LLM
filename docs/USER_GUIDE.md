## 概述

WorldEngine 是一个多角色世界模拟系统。你通过 Telegram 与一个虚拟世界互动，世界中有多个 AI 角色，他们有自己的记忆、情绪、日程和行为逻辑。你扮演其中一个角色，与其他角色对话、移动、打电话、参与事件。

系统包含三个组件：

| 组件 | 用途 | 访问方式 |
|------|------|----------|
| **聊天 Bot** | 沉浸式角色对话 | Telegram |
| **管理 Bot** | 世界控制命令 | Telegram（另一个 Bot） |
| **Dashboard** | 可视化世界管理 | 浏览器 `http://localhost:8080` |

## 第一步：安装与配置

### 1.1 环境要求

- Python 3.10+
- Windows（推荐）/ macOS / Linux
- 一个 **OpenAI 兼容格式**的 LLM API Key（如 [302.ai](https://302.ai)、OpenAI、DeepSeek、OpenRouter 等均可）
- 两个 Telegram Bot Token（通过 [@BotFather](https://t.me/botfather) 创建）

### 1.2 安装

```bash
git clone https://github.com/YOUR_USERNAME/WorldEngine.git
cd WorldEngine
pip install -r requirements.txt
```

### 1.3 配置

复制配置模板：

```bash
cp data/config.example.json data/config.json
```

编辑 `data/config.json`，填入以下必填项：

```json
{
  "api": {
    "base_url": "你的 API 地址（如 https://api.openai.com 或 https://api.302.ai）",
    "api_key": "你的 API Key"
  },
  "telegram": {
    "bot_token": "聊天 Bot 的 Token",
    "admin_bot_token": "管理 Bot 的 Token",
    "owner_id": 你的 Telegram 用户 ID
  }
}
```

> **`owner_id`** 是你的 Telegram 数字 ID，填写后只有你能使用 Bot。可以先不填，但建议尽快设置。获取方法：给 [@userinfobot](https://t.me/userinfobot) 发任意消息。

其他配置项保持默认即可，后续可以通过 Dashboard 的「设置」页面修改。

### 1.4 启动

```bash
# Windows 用户直接双击
start.bat

# 或命令行启动
python -m world_engine
```

启动后会自动打开浏览器访问 Dashboard。

## 第二步：创建你的世界

首次启动后，世界是空的。你需要通过 Dashboard 创建角色、地点和世界观。

### 2.1 设置世界基础信息

打开 Dashboard → **设置** 标签页：

1. 填写**世界名称**（如「云海市」）
2. 设置**起始时间**（如 `2026-01-01T08:00:00`）
3. 确认模型配置无误（默认即可）

### 2.2 创建地点

打开 Dashboard → **地图** 标签页：

1. 点击画布上的空白区域添加新地点
2. 输入地点名称（如「云海市中心公寓」「商业街」「大学」）
3. 拖拽图钉调整地点的相对位置（系统会根据坐标自动计算步行距离）
4. 为每个地点添加描述（点击地点名 → 编辑描述）
5. 可选：添加**子地点**（如「商业街」下有「咖啡店」「书店」「花店」）

> **位置关系很重要**：坐标决定了角色步行到其他地点的时间。把近的地方放近，远的放远。

### 2.3 创建角色

打开 Dashboard → **角色** 标签页：

1. 在底部输入角色名，点击「新建角色」
2. 为角色填写四个设定区：

| 区域 | 含义 | 示例 |
|------|------|------|
| **公开基础** | 外貌、职业等他人可知的设定 | 「24岁女生，在咖啡店打工」 |
| **私密基础** | 只有自己知道的设定 | 「暗恋隔壁的邻居」 |
| **公开动态** | 外在状态变化（系统自动管理） | 通常起步留空 |
| **私密动态** | 内心状态变化（系统自动管理） | 通常起步留空 |

3. 重复以上步骤创建所有角色（建议 2-5 个起步）
4. 为其中一个角色设定为**你扮演的角色**（在设置页面选择）

> **Tips**：
> - 「基础」区域是你手写的设定，系统不会修改
> - 「动态」区域由 AI 在对话过程中自动添加记忆，起步时可以留空
> - 每条设定单独一行，写得具体比笼统好

### 2.4 设置世界观（可选）

打开 Dashboard → **设置** 标签页 → 世界设定区域：

- **世界概述**：故事发生的背景（如「一个普通的现代都市」）
- **时代背景**：科技水平、社会环境
- **叙事基调**：你希望的故事风格（如「温馨日常」「悬疑推理」）
- **术语表**：世界中的关键概念（支持公开/秘密双层描述）

### 2.5 初始化角色位置

创建角色后，每个角色会被默认放在第一个地点。你可以：

- 通过管理 Bot 发送 `/move 角色名 地点名` 把角色分散到不同地点
- 或在 Dashboard 的设置页面调整

### 2.6 保存存档

世界搭建好之后，**务必保存一个存档**作为备份：

- Dashboard → **存档** 标签页 → 保存当前
- 或管理 Bot 发送 `/save 初始存档`

## 第三步：开始对话

### 3.1 发送第一条消息

在 Telegram 中找到你的**聊天 Bot**，直接发送消息即可开始。

系统会自动判断你当前位置有哪些角色在场，并让合适的角色回复你。

### 3.2 基本对话方式

| 操作 | 方法 | 示例 |
|------|------|------|
| 普通对话 | 直接发消息 | `你好啊，今天天气真好` |
| 指定对某人说 | `@角色名` 开头 | `@小美 你今天怎么样？` |
| 描述动作 | 用括号包裹 | `（走到窗边，看着外面）好无聊啊` |
| 触发移动 | 自然地说去某地 | `那我们去咖啡店坐坐吧` |
| 打电话 | `/call 角色名` | `/call 小美` |
| 挂断电话 | 说"挂了"或 `/hangup` | `好，先挂了` |

### 3.3 理解消息格式

每次对话后，系统会在回复末尾附上当前状态：

```
🕐 01-15 14:30 周三 · 📍 商业街/咖啡店
```

可能出现的特殊消息：

| 标记 | 含义 |
|------|------|
| 🎲 | DM 裁定了你的动作结果 |
| 📖 | 环境旁白（描述场景氛围） |
| 📤 | 有角色离开了当前场景 |
| ⏳ | 时间大幅跳转的过渡描述 |

## Dashboard 详细说明

访问 `http://localhost:8080`，包含以下标签页：

### 会话

查看所有对话记录（活跃的和已归档的）。每个会话显示参与者、地点和消息内容。

### 事件

分为两部分：
- **预定事件**：角色之间约定的计划（如「明天下午两点在咖啡店见面」）。系统会自动检测对话中的约定并创建事件，也可以手动添加
- **世界事件**：已发生的事件记录

### 角色

编辑角色的四个设定区域。每条设定有 TTL（存活时间）标记：
- **永久**：永不过期（基础设定用）
- **Nh**：N 小时后过期（如 `24h`）
- **Nm**：N 分钟后过期

### 地图

2D 地图编辑器：
- 拖拽图钉调整地点位置
- 点击地点编辑名称、描述
- 管理子地点

### 存档

- **保存当前**：把当前世界状态保存为快照
- **加载存档**：恢复到某个存档
- **浏览存档**：查看存档内容但不加载
- **新建空白**：创建全新的空白世界

### 设置

- 模型配置（对话、分析、嵌入、图片生成）
- 世界参数（步行速度、压缩阈值等）
- 世界设定（Lore）
- 费用价格配置

## 核心概念

### 虚拟时间

世界有自己的时间线。**时间不会自动流逝**，只有当你发消息时，AI 才会根据对话内容判断推进多少时间（通常几分钟）。使用 `/time` 命令可以手动快进。

时间推进时，不在你身边的角色也会自动安排活动（去不同的地点、做不同的事），这让世界感觉是"活着"的。

### 场景与位置

- 同一地点的角色可以面对面互动
- 不同地点的角色需要通过 `/call` 打电话
- 说"去某地"会触发移动，AI 会自动处理
- 移动需要时间（根据地点距离计算步行时间）

### 记忆系统

角色会自动记住对话中的重要信息。记忆分为：
- **公开记忆**：其他角色知道的事实（如「小美说她喜欢画画」）
- **私密记忆**：只有自己知道的想法（如「我觉得他在隐瞒什么」）

记忆有**过期时间**（TTL），不重要的记忆会自然遗忘，重要的记忆会被标记为永久。相似的记忆会被自动合并。

### DM 系统

一个隐形的"主持人"（Dungeon Master）在每次对话前运行：
- **裁定动作**：你用括号写的动作不一定会成功，DM 会根据合理性判断结果
- **环境旁白**：描述场景氛围、路人反应等
- **管理临时角色**：没有角色文件的 NPC（如店员、路人）由 DM 管理

### 预定事件

角色在对话中做出的约定（如「明天下午见」）会被系统自动记录。到约定时间时，角色会自行判断是否赴约。如果有人爽约，被放鸽子的角色会获得相应记忆。

## 进阶技巧

### 世界构建建议

1. **角色设定要具体**：「24岁，在咖啡店打工，梦想成为画家」比「年轻女性」好得多
2. **设定内在矛盾**：角色有矛盾才有故事（如「外表开朗但内心自卑」）
3. **地点要有功能**：每个地点应该有让角色去的理由（家、工作地、休闲场所）
4. **关系要有种子**：在设定中埋下关系伏笔（如「是 XX 的邻居」「和 XX 是大学同学」）

### 调试与回溯

- **Turn 日志**：`logs/turns/` 目录下有每轮交互的详细 HTML 调试页面，记录了所有 LLM 调用的完整 Prompt、回复和费用
- **引擎日志**：`logs/engine.log` 记录了所有事件的流水账
- **勤存档**：重要节点前保存存档，出问题随时回档

### 费用控制

在 Dashboard 设置页配置 `model_pricing` 后，每轮交互的 Turn 日志会自动显示 LLM 调用费用明细。

## 常见问题

### 启动后 Bot 没有响应？

1. 检查 `data/config.json` 中的 `bot_token` 和 `admin_bot_token` 是否正确
2. 检查 `owner_id` 是否填对（必须是你的 Telegram 数字 ID）
3. 查看终端输出是否有错误信息

### 角色不回复？

- 确认你和要对话的角色在**同一地点**（用 `/where` 查看）
- 如果不在同一地点，用 `/call 角色名` 打电话
- 用 `@角色名` 指定对谁说话

### 如何重置世界？

1. 如果有保存存档：`/load 存档名` 或 Dashboard → 存档 → 加载
2. 如果要完全重来：Dashboard → 存档 → 新建空白世界

### Dashboard 打不开？

- 默认地址是 `http://localhost:8080`
- 检查终端是否显示 Dashboard 启动成功
- 确认端口 8080 没有被其他程序占用
