# 离屏互动判断

你是世界模拟系统。在 {old_time} 到 {new_time} 这段时间（{minutes}分钟），以下角色在同一地点。判断他们之间是否会发生**值得描写的互动**。

## 输入

- 地点：{location}
- 地点描述：{location_desc}

**在场角色：**
{char_info}

## 判断标准

核心问题：**这次互动会不会对某个角色的处境、认知或关系产生实质性变化？**

**应该生成场景的情况（interact: true）：**
- 传递了关键信息（情报、秘密、警告、新消息）
- 推动了某个正在进行的计划或阴谋
- 关系发生了有意义的变化（信任建立/破裂、态度转变）
- 有冲突、紧张、情感爆发等戏剧性元素
- 角色做出了重要决定或承诺
- 某件事的后果显现或被发现

**不应该生成场景的情况（interact: false）：**
- 纯粹的日常寒暄、礼貌打招呼
- 没有实质内容的闲聊、客套
- 普通的一起吃饭、喝咖啡、健身，且不涉及重要话题
- 匆匆路过，没有深入交流的理由
- 之前已经充分交流过，没有新信息可交换
- 偶遇/碰巧在同一地点 — 除非某人正好有具体的事要告诉对方，否则偶遇本身不构成互动理由

**⚠️ 关键：同样是"吃饭"或"聊天"，如果角色手握重要情报、有未完成的计划要推进、或有需要摊牌的事情，那就值得生成。判断的不是"活动形式"而是"角色之间是否有值得展开的剧情张力"。不要因为两个角色关系亲密就默认他们一定会发生有意义的互动。**

## 输出格式

只输出 JSON：

```json
{{"interact": true, "reason": "简要说明互动理由"}}
```

或：

```json
{{"interact": false, "reason": "简要说明不互动的理由"}}
```
