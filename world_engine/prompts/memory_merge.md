# 记忆合并

以下是角色 "{character}" 的动态记忆条目（已排除永久记忆）。每条标注了所属板块（公开动态/私密动态）和当前 TTL。找出可以合并的条目。

## 输入

{items}

## 规则

### 可以合并的情况

- **同一天同一时段、同一事件的多条碎片**：比如"早上和A吵架"和"早上吵完架后打电话道歉"明显是同一事件的不同阶段，必须合并
- **内容高度重叠的记忆**：只是措辞不同的同一件事
- **过时的早期版本**：如果有一条更完整的记忆已经覆盖了旧内容，旧的可以被删除

### 绝对不能合并的情况

- **不同天的记忆**：1月1号的事和1月2号的事不能合并
- **同一天但不同时段、不同事件**：比如"早上买东西"和"下午去学校上课"，虽然是同一天但是完全不同的事
- **不同板块**：公开动态只能和公开动态合并，私密动态只能和私密动态合并
- **因果上无关的记忆**：两件事虽然时间接近但没有因果关系，不要合并

### 合并质量要求

- **合并后的内容必须保留所有关键信息**，不能丢细节。宁可合并后的句子长一些，也不要遗漏重要内容
- 合并后的 TTL 取合并前各条中最长的那个
- 如果不确定是否该合并，**倾向于不合并**——宁可多留几条，也不要错误合并导致信息丢失
- 如果没有需要合并的，返回空列表

## 输出格式

只输出 JSON：

```json
{{"merges": [{{"keep_indices": [保留的序号], "remove_indices": [删除的序号], "merged_content": "合并后的内容", "ttl": "合并后的TTL"}}]}}
```
